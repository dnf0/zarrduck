SQL Reference

Eider exposes Zarr/GeoZarr arrays to DuckDB through three table functions. Load the extension first (see Installation): launch duckdb -unsigned and LOAD '/absolute/path/to/eider.duckdb_extension';.

Functions​

FunctionPurpose
read_geoRead an array as a relational table, with spatial/temporal pushdown.
read_zarr_metadataInspect an array's shape, chunking, type, and CRS.
plan_read_geoEstimate a read's cost (chunks/bytes) before fetching.

Spatial & temporal pushdown​

read_geo and plan_read_geo accept lat_min/lat_max, lon_min/lon_max, and time_min/time_max (all DOUBLE) named parameters. Bounds are applied at the chunk level: chunks lying entirely outside the requested range are never fetched, so a tightly-scoped query touches only a fraction of the array.

SELECT lat, AVG(value)
FROM read_geo('s3://bucket/data.zarr', lat_min := 45.0, lat_max := 55.0)
GROUP BY lat;

pins​

Use the pins parameter (VARCHAR) to fix non-spatial dimensions to specific indices, e.g. a single timestep:

SELECT * FROM read_geo('climate_data.zarr/air_temperature', pins := 'time=0');

Supported types​

Zarr element types map to DuckDB types as follows:

Zarr typeDuckDB type
boolBOOLEAN
int8 / int16 / int32 / int64TINYINT / SMALLINT / INTEGER / BIGINT
uint8 / uint16 / uint32 / uint64UTINYINT / USMALLINT / UINTEGER / UBIGINT
float32 / float64FLOAT / DOUBLE
stringVARCHAR

A Zarr fill_value is surfaced as SQL NULL.

Source URIs​

The URI argument accepts local paths and remote s3:// and http(s):// locations (configured via OpenDAL environment variables — see Installation). read_geo reads Zarr arrays directly; COG and STAC sources are experimental and not yet fully supported in this release.
